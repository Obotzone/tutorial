SEO Dasar untuk Cloudflare Pages

Cloudflare Pages sangat cepat dan stabil, namun tetap perlu pengaturan SEO agar halaman kamu mudah terindex Google, mendapatkan ranking, dan dipercaya mesin pencari.

1. Gunakan Meta Title & Description yang Tepat

Setiap halaman wajib punya:

<title>Judul halaman (maks 55 karakter)</title>
<meta name="description" content="deskripsi 150–160 karakter">

Di Cloudflare Pages tidak ada backend, jadi kamu harus menulis ini secara manual di setiap HTML.

Meta title & description adalah kunci pertama yang dibaca Google.

2. Gunakan URL bersih (tanpa query)

Untuk SEO, hindari URL seperti:

/halaman.html?id=123

Pakai nama file seperti:

/panduan-cloudflare-pages.html
Google lebih suka URL jelas dan pendek.

3. Struktur konten rapi: H1, H2, H3

Gunakan format heading yang rapi:

  • H1 → Judul utama
  • H2 → Subjudul
  • H3 → Penjelasan detail

Cloudflare Pages tidak punya CMS otomatis, jadi heading manual adalah hal wajib.

4. Tambahkan Sitemap XML

Buat file:

/sitemap.xml

Contoh isi:

<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://domainkamu.com/</loc>
  </url>
  <url>
    <loc>https://domainkamu.com/tutorials/</loc>
  </url>
</urlset>
Setelah membuat sitemap, submit di Google Search Console.

5. Tambahkan robots.txt

Buat file:

/robots.txt

Isi minimalnya:

User-agent: *
Allow: /
Sitemap: https://domainkamu.com/sitemap.xml

6. Daftarkan Domain ke Google Search Console

Ini langkah paling penting untuk index cepat.

  • Buka: https://search.google.com/search-console
  • Pilih “URL Prefix”
  • Masukkan domain/subdomain kamu

Lalu verifikasi menggunakan:

  • HTML tag (paling mudah)
  • DNS TXT Record
Setelah terverifikasi, klik: Inspect URL → Request Indexing

7. Kecepatan = poin SEO lebih tinggi

Cloudflare Pages sudah sangat cepat, namun optimasi ini bisa menambah nilai:

  • Kompres gambar sebelum upload
  • Gunakan loading="lazy" untuk gambar
  • Minify CSS/JS (kalau ada file besar)
  • Gunakan ukuran font standar

8. Mobile-first (responsif)

Google memakai Mobile-First Indexing. Gunakan layout responsif agar halaman tampil bagus di smartphone.

Jika halaman tidak responsif → ranking otomatis turun drastis.

9. Hindari konten duplikat

Jika kamu menggunakan banyak subdomain digitalplat.org, pastikan setiap halaman berisi konten berbeda.

Google akan menurunkan ranking jika semua halaman sama.

10. Gunakan internal link

Hubungkan halaman kamu satu sama lain:

  • Home → Tutorials
  • Tutorial #1 → #2 → #3
  • Footer link ke halaman penting
Mesin pencari lebih cepat meng-crawl website yang saling terhubung.

11. Tingkatkan “Authority” situs kamu

Untuk exposure Obotzone, metode berikut sangat efektif:

  • Gunakan subdomain berkualitas (digitalplat.org sudah bagus)
  • Jangan gunakan hosting murahan/bermasalah
  • Perbanyak konten bermanfaat (tutorial, panduan)
  • Promosikan melalui sosial media (Telegram / X)
Semakin besar exposure → semakin mudah semua subdomain kamu naik ranking.

12. Setelah SEO dasar → kita masuk SEO lanjutan

Di tutorial berikutnya kita akan belajar:

  • Structured Data / Schema JSON-LD
  • Meta OG untuk share preview
  • PageSpeed Insights tuning
  • Internal link strategy untuk jaringan subdomain
