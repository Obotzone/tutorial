Struktur Project untuk Cloudflare Pages

Setelah mengenal konsep Cloudflare Pages, kini kamu belajar cara menata folder dan file supaya website statis kamu siap untuk proses deployment yang mulus.

1. Struktur folder dasar

Cloudflare Pages bekerja paling optimal jika struktur project kamu rapi dan konsisten seperti berikut: 

/
├─ index.html
├─ about.html   (opsional)
├─ tutorials/   (folder halaman tutorial)
│   └─ index.html
├─ assets/
│   ├─ css/
│   │   └─ style.css
│   ├─ img/
│   └─ js/
└─ README.md (opsional)

index.html wajib ada dan menjadi halaman utama. Folder assets menyimpan semua gambar, css, dan script.

2. Memisahkan landing page & tutorial

  • /index.html → landing page utama Obotzone
  • /tutorials/ → semua halaman tutorial
  • Setiap tutorial punya 1 file HTML sendiri

3. Kenapa struktur ini penting?

  • Memudahkan Cloudflare membaca file statis.
  • Memudahkan kamu update satu halaman tanpa merusak halaman lain.
  • URL menjadi lebih rapi dan SEO-friendly.

4. Pastikan project siap deploy

Checklist sebelum deploy:

  • Ada file index.html di root.
  • Ada folder /assets/css/style.css.
  • Link relatif benar, contoh: href="/assets/css/style.css"
  • Folder tutorial lengkap & link-nya tidak error.
Jika semua sudah rapi → project siap dideploy ke Cloudflare Pages tanpa error.
