Mengenal Cloudflare Pages & Kenapa Cocok Untuk Website Ringan

Sebelum mulai klik-klik panel Cloudflare, kamu perlu paham dulu: apa itu Cloudflare Pages, apa yang dia lakukan untuk websitemu, dan kenapa ini menarik kalau kamu ingin bangun exposure Obotzone + manfaatkan subdomain gratis dari digitalplat.org.

Durasi baca ± 5 menit
Target: pemula tapi pengen naik level

1. Apa itu Cloudflare Pages?

Cloudflare Pages adalah layanan hosting gratis untuk website statis (HTML, CSS, JS) yang berjalan di infrastruktur Cloudflare. Bahasa sederhananya: kamu naruh file websitemu di sebuah repo (misalnya GitHub), lalu Cloudflare Pages yang meng-online-kan dan meng-cache website kamu di banyak server di seluruh dunia.

Gambaran alur singkat

1) Kamu push/update kode website → 2) Cloudflare Pages otomatis build & deploy → 3) Website langsung update di domain / subdomain yang kamu pakai.

Ini yang bikin cocok untuk sistem subdomain gratis: satu panel, bisa banyak project, dan deployment tanpa ribet upload manual.

2. Kenapa cocok untuk sistem subdomain & exposure?

Ada beberapa hal yang bikin Cloudflare Pages menarik kalau kamu mau bermain banyak website ringan dengan subdomain:

Keunggulan (Pro)

  • Gratis & scalable – Bisa mulai tanpa biaya, cocok untuk eksperimen banyak landing page.
  • Cepat – File di-cache di edge server Cloudflare, jadi pengunjung dari mana pun relatif cepat mengakses.
  • Integrasi mudah dengan DNS Cloudflare – Cocok untuk skema subdomain dari penyedia seperti digitalplat.org.
  • Deployment otomatis – Setiap commit ke repo bisa langsung jadi deploy baru.

Keterbatasan (Con)

  • Statis – Tidak ada PHP/SQL langsung; cocoknya untuk landing page, dokumentasi, profil, dan web informatif.
  • Butuh repo – Biasanya terhubung ke Git (GitHub, GitLab, dsb), jadi sedikit butuh adaptasi di awal.
  • Bukan untuk aplikasi berat – Untuk aplikasi kompleks, perlu kombinasi dengan layanan lain.

Justru karena fokus di website ringan dan informatif, kombinasi Cloudflare Pages + subdomain gratis pas untuk bangun jaringan exposure dan testing ide sebelum di-upgrade ke domain utama.

3. Hubungannya dengan subdomain gratis dari digitalplat.org

digitalplat.org berperan sebagai penyedia subdomain gratis. Artinya, kamu bisa punya alamat seperti:

  • namaproject.salahsatu-extension.digitalplat.org
  • Setiap project bisa diarahkan ke satu project di Cloudflare Pages.

Dengan setup yang tepat, kamu bisa punya banyak situs:

  • Satu untuk exposure Obotzone.
  • Beberapa untuk tutorial dan panduan teknis.
  • Beberapa lagi untuk eksperimen landing page khusus niche tertentu.
Nilai tambah saat sudah punya exposure

Kalau website kamu konsisten bermanfaat, punya traffic, dan sering direkomendasikan, nilai brand-mu ikut naik:

  • Subdomain jadi punya value brand, bukan cuma alamat gratisan.
  • Setiap subdomain baru ikut “nebeng” reputasi jaringan yang sudah dikenal.
  • Kalau nantinya di-upgrade ke domain utama berbayar, asset konten & trafik sudah siap.

Inilah kenapa penting membangun fondasi yang rapi sejak awal, meski “hanya” pakai subdomain.

4. Yang kamu butuhkan sebelum lanjut ke langkah teknis

Sebelum masuk ke tutorial berikutnya (yang mulai klik-klik panel Cloudflare Pages), pastikan kamu sudah siapkan:

  • Akun Cloudflare.
  • Minimal satu repo berisi file website statis (HTML + CSS saja pun sudah cukup).
  • Akses ke pengelolaan DNS untuk subdomain dari digitalplat.org (nanti diatur mengarah ke project Cloudflare Pages).

Tenang, di tutorial-tutorial berikutnya kita akan buat semua langkah ini satu per satu, dari setup paling dasar sampai siap dipakai banyak subdomain.

Selanjutnya: kita akan mulai dari sisi Cloudflare Pages – membuat project pertama yang terhubung ke repo.
