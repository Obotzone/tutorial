Tutorial #1

Pengenalan Cloudflare Pages & Persiapan Akun

Di tutorial pertama ini kamu akan memahami dulu apa itu Cloudflare Pages, kenapa cocok untuk website statis ringan, dan apa saja yang perlu disiapkan sebelum mulai deploy.

Yang akan kamu pelajari:
  • Apa itu Cloudflare Pages dan kelebihannya.
  • Perbedaan Cloudflare Pages dengan hosting klasik.
  • Apa saja yang harus kamu siapkan (akun, file, dan domain).
  • Gambaran alur kerja sebelum masuk tutorial teknis berikutnya.

1. Apa itu Cloudflare Pages?

Cloudflare Pages adalah layanan hosting untuk website statis yang berjalan di jaringan global Cloudflare. Artinya:

  • File HTML, CSS, dan JavaScript kamu disimpan di edge server Cloudflare.
  • Pengunjung akan mengakses website dari lokasi server terdekat (lebih cepat).
  • Tidak perlu pusing dengan server, SSH, atau panel hosting yang rumit.

Cloudflare Pages sangat cocok untuk:

  • Landing page ringan seperti halaman Obotzone.
  • Website profil, portfolio, dokumentasi, atau blog statis.
  • Web tutorial dan halaman informasi yang jarang berubah.

2. Kelebihan Cloudflare Pages dibanding shared hosting biasa

  • Cepat – Menggunakan jaringan global (CDN) Cloudflare.
  • Gratis – Untuk skala kecil–menengah, tier gratis sudah sangat cukup.
  • Mudah – Cukup unggah file atau sambungkan ke Git, tanpa setting server.
  • HTTPS otomatis – Sertifikat SSL dikelola langsung oleh Cloudflare.
  • Skalabel – Website tetap stabil saat trafik mendadak naik.

3. Kekurangan & batasan yang perlu kamu tahu

Meski enak untuk landing page dan website statis, Cloudflare Pages juga punya batasan:

  • Hanya untuk konten statis – Tidak ada PHP, tidak ada database yang berjalan langsung di server (untuk fitur dinamis butuh layanan tambahan).
  • Butuh penyesuaian pola pikir – Dari “hosting klasik” ke model Jamstack / static site (file build & deploy).
  • Beberapa fitur advanced (misalnya backend API) perlu integrasi dengan layanan lain (Workers, KV, D1, dsb).

Untuk kebutuhan landing page, profil, dan tutorial – Cloudflare Pages biasanya sudah lebih dari cukup. Fokus kita di seri tutorial ini adalah memaksimalkan kekuatan website statis dulu.

4. Apa yang perlu disiapkan sebelum mulai?

Sebelum lanjut ke langkah teknis, pastikan kamu sudah punya beberapa hal ini:

  • Akun Cloudflare – Satu akun utama yang akan mengelola semua project dan domain kamu.
  • File website statis – Minimal berisi index.html, style.css, dan aset pendukung (gambar, ikon, dsb).
  • Domain (opsional, tapi disarankan) – Bisa domain utama, subdomain proyek, atau subdomain dari penyedia seperti digitalplat.org.

Kalau kamu belum punya domain sendiri, kamu tetap bisa mulai dulu menggunakan domain default dari Cloudflare Pages. Nanti saat sudah siap, baru dihubungkan ke domain atau subdomain yang kamu inginkan.

5. Gambaran alur yang akan kita pakai

Supaya kamu punya “peta besar” sebelum lanjut, kurang lebih alur penggunaan Cloudflare Pages dalam seri tutorial ini seperti berikut:

  • Pahami konsep & siapkan akun Cloudflare.
  • Buat project baru di Cloudflare Pages. (upload file atau sambungkan ke Git)
  • Atur build & deployment. (kalau pakai generator atau framework tertentu)
  • Hubungkan ke domain atau subdomain pilihanmu.
  • Optimasi & kembangkan. (struktur folder, update konten, tambah halaman tutorial, dsb)

Di tutorial #1 ini kamu cukup paham dulu di level konsep. Mulai tutorial #2, kita akan langsung praktik: membuat akun Cloudflare dan melihat menu Pages langkah demi langkah.
